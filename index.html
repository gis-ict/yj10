SHEET 01NEW BUILD · STORAGE RACK

The List — Storage Rack

A list holds many items in order, inside square brackets [ ]. It can change any time — add, read a slot, or count what's inside. This is your everyday container.

COMPONENT SPEC · list [ ]
lists.py 
team = ["Mia", "Sam", "Lee"]

print(team[0])       # Mia → slots start at 0
print(len(team))     # 3 → how many items

team.append("Zoe")  # add to the END
team[1] = "Max"     # change a slot (lists can change!)
print(team)          # ['Mia','Max','Lee','Zoe']
[ ] = listslot 0 = first.append() = addcan change
Real build: a list of players, a shopping cart, scores in order — anywhere the order matters and the contents change.

SHEET 02NEW BUILD · LOCKED PART

The Tuple — Locked Part

A tuple looks like a list but uses round brackets ( ) and is locked. Once built, it cannot change. Use it for data that should never be edited by accident.

COMPONENT SPEC · tuple ( )
tuples.py 
point = (10, 20)     # round brackets = tuple

print(point[0])     # 10 → you can still READ slots
print(len(point))   # 2

point[0] = 5      # ❌ ERROR! tuples are LOCKED
( ) = tupleread OKchange = errorstays safe
List vs Tuple: same idea, different bracket. [ ] can change, ( ) is frozen. Pick a tuple when the values must stay fixed — like map coordinates or days of the week.

SHEET 03NEW BUILD · UNIQUE BIN

The Set — Unique Bin

A set uses curly brackets { } and keeps only unique items — no duplicates allowed. Add the same thing twice and the second one just disappears.

COMPONENT SPEC · set { }
sets.py 
fruits = {"apple", "banana", "apple"}
print(fruits)        # {'apple','banana'} → duplicate gone!
print(len(fruits))   # 2 → only unique items counted

fruits.add("orange") # add a new unique item
fruits.add("apple")  # ignored — already there
{ } = setno duplicates.add()unordered
Real build: a list of unique usernames, or every different country a class has visited — anywhere a thing should appear once only.

SHEET 04NEW BUILD · LABELLED DRAWERS

The Dictionary — Labelled Drawers

A dictionary stores key : value pairs in curly brackets { } — like labelled drawers. You don't count to a slot; you open the drawer by its label.

COMPONENT SPEC · dict { key : value }
dictionaries.py 
player = {
    "name": "Ninja",    # label : value
    "level": 12
}

print(player["name"])   # Ninja → open by label
player["level"] = 13     # change a drawer
player["clan"] = "Shadow" # add a new drawer
key : valueopen by labelcan changemodels one object
Set vs Dict: both use { }. A set holds single items {"a","b"}; a dict holds pairs {"a":1}. The colon is the giveaway.

SHEET 05NEW BUILD · REUSABLE MODULE

The Function — Reusable Module

A function is a part you design once and reuse forever. Build it with def, give it inputs, and call it whenever you need it. This is the heart of building bigger programs.

COMPONENT SPEC · def name(input)
functions.py 
# DEFINE the module once
def greet(name):
    print(f"Hello, {name}!")

# CALL it as many times as you like
greet("Alex")
greet("Mia")

# RETURN sends an answer back out
def add(a, b):
    return a + b

total = add(4, 3)    # total = 7
def = define( ) = inputsreturn = answerbuild once, reuse
Mental model: a function is a microwave button. Wire up the logic once, then just press it. The inputs are the food; return is the meal you get back.
